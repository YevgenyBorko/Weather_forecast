Execution

The submission is a WebStorm project that can be run directly from the IDE.

The project uses a server and client model to fetch responses from the server to user requests

Each Request from the user is taken care of by The Server, even the incorrect ones

The Data from users is stored in SQLite Tables

There are 2 tables defined in the project, one stores the users, another stores user data per user.


We loaded 2 images from the weather server into the top right division and made separate divisions for each day

Also we added status display for selected coordinates in dropdown menu.


For user validation i created divisions under each field so each field gets it's corresponding errors

There was an addition of Clear Button to the user wiev which clears all stored data in that user table

and clears the dropdown menu as well



Assumptions

The site use bootstrap CDN therefore assumes an internet connection is available.
